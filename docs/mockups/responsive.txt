================================================================================
RESPONSIVE BEHAVIOR — Terminal Size Adaptations
Assumes: Phosphor theme (green-on-black)
================================================================================

Shows how the UI adapts to different terminal sizes. The app remains functional
at any size but optimizes layout for common dimensions.

COLOR KEY:
  [green]   = phosphor green #00d700 (ANSI 40) — primary text
  [cyan]    = bright cyan #00ffff (ANSI 51) — interactive accent
  [dim]     = dark green #005f00 (ANSI 22) — borders, decorative, structural
  [med]     = medium green #00af00 (ANSI 34) — secondary text
  [white]   = warm off-white #e4e4e4 (ANSI 254) — selected item text
  [sel-bg]  = dark green bg (ANSI 22) — selection background

                                                        [green]exploration.[/]

                                                        [green]The two-stage rocket lifted off from SpaceX's Starbase[/]

                                                        [green]on the thrust of 33 Raptor engines that generate roughly[/]
                                                        [green]16.7 million pounds of force at full power.[/]

NOTES:
- Reader content is always centered in the terminal
- Left margin = (terminal_width - MaxContentWidth) / 2
- MaxContentWidth default is 80 (user-adjustable with h/l)
- This prevents the ultra-long line lengths that make text hard to read
- Status bar content aligns to the content column, not terminal edges


================================================================================
DESIGN PRINCIPLES
================================================================================

1. Content max-width: ~120 chars for multi-column layouts, ~80 for single-column
   reader text. Never let content span the full width on very wide terminals.
2. Centering: On wide terminals, center the content area with equal margins.
3. Progressive disclosure: Remove UI elements as space shrinks:
   - First: reduce hint tiers (full → medium → minimal)
   - Then: remove separators between items (height < 30)
   - Then: compact card height (height < 15, 5-line → 3-line)
   - Then: single-column mode (width < threshold)
   - Then: inline CTA instead of slab CTA (height < 20)
4. Minimum supported: 50 columns x 15 rows. Below this, the app remains
   functional but visual quality degrades (truncated titles, no separators,
   minimal hints).
