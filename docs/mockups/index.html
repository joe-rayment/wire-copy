Screens

  • Launcher · spec
    Home screen with bookmarks grid/list. Entry point to all navigation.
  • Link Tree · spec
    Hierarchical view of all links on a loaded page. Card grid or detailed list.
  • Reader · spec
    Focused article reading view. Centered column with adjustable width.
  • Collections · spec
    Reading list / collection management and organization.
  • Reading List · spec
    Saved articles in a collection. Podcast generation CTA.
  • Status Bar · spec
    Adaptive 2-line footer: mode + hints, then context vitals.
  • Toasts · spec
    Overlay notifications: info, success, error, celebration, undo.
  • Help Overlay · spec
    Context-sensitive keybinding reference panel.
  • Loading & Errors · spec
    Network states: loading, error, bot challenge, login. Esc to cancel.
  • Animations · spec
    Storyboard frames for transitions and celebrations.
  • Layout Switcher · spec
    Ctrl+L layout cycling overlay and persistence.
  • Responsive · spec
    Terminal size adaptation: 60x16, 80x24, 200+ cols.

Design Principles

1. Reading comfort first Body text is pure phosphor green (#00d700), vivid but not neon. Line height 1.6. Centered columns default to 66 chars. Titles in playful pink (#ff87d7) create instant hue contrast against body text. 2. Green reads, pink leads The warm green body text is for reading. The pink is for navigation: titles, headers, emphasis. The hue contrast between green and pink is unmistakable — no squinting at similar shades of green. 3. Chrome serves content Headers are single-line. Separators are omitted where blank lines suffice. Status bar is 2 lines. Every line of chrome must justify the content it displaces. 4. Cyan means interactive Cyan (#00ffff) is reserved exclusively for elements the user can press or activate. Status indicators use --med or --muted instead. 5. Delight without friction Animations never block the hot path (navigation, scrolling). Celebrations are earned (podcast complete), not routine. Micro-feedback (mode flash, save count) costs zero latency. 6. Errors are recoverable Esc cancels loading. Destructive actions get undo toasts. Single-keypress deletes are reversible for 5 seconds.
