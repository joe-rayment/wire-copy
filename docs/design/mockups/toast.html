Positioning
Toasts appear at row 0, right-aligned with 1-char margin from right edge.
Only one toast is visible at a time. New toasts replace the current one immediately. No stacking.
Dismissal Behavior
Appear: Instant (no slide-in animation — terminal rendering is frame-based).
Display: Shown on the current render pass.
Dismiss: Cleared on the next render pass (triggered by next keypress or redraw).

Standard toasts (confirmation, info): Dismissed automatically on next keypress or screen redraw.
Sticky toasts (error, celebration): Persist across redraws until user presses Esc. Show a small [Esc] hint inside the toast border.
Undo toasts (removal): Auto-dismiss after 5 seconds. Pressing z undoes the action before timeout.

Any keypress (including Esc for sticky) also processes normally — the toast dismiss is a side effect, not a blocking modal.
