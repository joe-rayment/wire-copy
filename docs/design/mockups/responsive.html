Design Principles
1. Content max-width: ~120 chars for multi-column layouts, ~66 for single-column reader text. Never let content span the full width on very wide terminals. 2. Centering: On wide terminals, center the content area with equal margins. 3. Progressive disclosure: Remove UI elements as space shrinks: - First: reduce hint tiers (full → medium → minimal) - Then: remove separators between items (height < 30) - Then: compact card height (height < 15, 5-line → 3-line) - Then: single-column mode (width < 80) - Then: inline CTA instead of slab CTA (height < 20) 4. Minimum supported: 50 columns x 15 rows. Below this, the app remains functional but visual quality degrades (truncated titles, no separators, minimal hints).
WHY: Progressive disclosure removes UI as space shrinks: hints → separators → card height → columns. Content always wins over chrome.
