This is what the user sees when the orchestrator returns a typed
PodcastResult.Failure(detail) with FailureClass=FeedNotReachable.
PodcastCommandHandler passes result.FailureDetail to ShowErrorScreenAsync,
which calls PodcastFailureClassifier.Classify(detail, ...) to produce the
(Step, Reason, Fix) tuple, then renders the layout below.

The rendering function (ShowErrorScreenAsync) is deterministic — given the
same typed FailureDetail, the on-terminal output is the same as this dump.
Pinned by PodcastFailureClassifierTests +
CompletionScreenShapeTests.ShapeD_BucketNotPublic_RendersBucketPublicRemediation
+ this visual smoke.
