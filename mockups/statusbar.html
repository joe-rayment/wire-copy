TermReader Mockups: Status Bar States
Status bar occupies bottom 2 lines: mode label + key hints, contextual vitals · Phosphor 2.0
Line 2 Indicator Priority
Priority order (left to right, truncate from right):
1. Context (always): domain or item count
2. Active state (when relevant): search query OR caching OR paywall
3. Passive (space permitting): layout badge, cache age
4. Position (reader only): L##/## ##%
State 1: Launcher Mode
Launcher Enter:open o:go to url a:add d:delete ?:help
Mode label "Launcher" in .g (green, primary text)
Keys in .c (cyan, interactive), descriptions in .m (med, metadata)
No line 2 content (no URL, no cache, no search)
Separator line removed — 2-line status bar, not 3
State 2: Launcher Mode — Narrow Terminal (hints adapt)
Launcher Enter:open o:go to url ?:help
Launcher ?:help
Hint tiers shrink as terminal narrows
Full tier: Enter, o, a, d, ?
Medium tier: Enter, o, ?
Minimal tier: ?
State 3: Link Tree Mode — with Cache Progress
LinkTree Enter:open s:save A:save-all R:refresh v:reader ?:help arstechnica.com ████████▏▏▏ 17/24 cached
Line 1: mode label in .g, key hints (keys .c, descriptions .m)
Line 2 left: domain in .m (metadata)
Line 2 right: eighth-block progress bar in .mu (muted, informational)
Cache progress uses filled blocks (█) and thin empty blocks (▏)
Fraction "17/24" and "cached" label in .mu

"caching" instead of "cached" label

Slug extracted from URL path (readable, truncated to ~30 chars)
