================================================================================
HELP OVERLAY — Keybinding Reference
Assumes: 80x24 terminal, Phosphor theme (green-on-black)
================================================================================

Appears when the user presses ? on any screen. Shows all keybindings for
the current view, grouped by category.

COLOR KEY:
  [green]   = bright green #00ff00 (ANSI 46) — category headers
  [cyan]    = bright cyan #00ffff (ANSI 51) — key characters
  [dim]     = dark green #005f00 (ANSI 22-28) — borders, decorative
  [med]     = medium green #00af00 (ANSI 34) — action descriptions

================================================================================
DESIGN NOTES
================================================================================

- Overlay is centered both horizontally and vertically on screen
- Box width: ~60 chars (fits in 70-column terminals with margins)
- All box corners are rounded: ╭ ╮ ╰ ╯ (never sharp ┌ ┐ └ ┘)
- Category headers in [green] (PrimaryText) for visual grouping
- Key characters in [cyan] (AccentFg) — consistent with all key hint styling
- Action descriptions in [med] (SecondaryText) — dimmer, non-interactive
- Footer "Press ? or Esc to close" uses same [cyan] key / [med] text pattern
- The overlay does NOT dim the background — it simply overwrites the center
  region and is cleared on dismiss (next ? or Esc press triggers full redraw)
- The help content is context-sensitive: it shows bindings for the CURRENT view
- Selection bar ▌ is not shown in the help overlay (it's not a navigable list)
